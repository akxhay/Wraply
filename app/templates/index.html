# Overview

API Wrapper Admin lets you store upstream API configurations in a SQLite database and execute them by posting a payload. Every aspect of the request — URL, headers, query string, body, and response — can be transformed dynamically using dot-path or JMESPath expressions resolved against the incoming payload.

📥
Your App sends POST /execute/:id { payload }
The call payload is the source of truth for all transforms
⚙️
Resolve all transforms against payload
Path params → URL  ·  Query params → ?string  ·  Body → request body  ·  Headers → headers
🌐
Call upstream API
Fully resolved URL, headers, auth, and body are sent
🔄
Apply response transform
Map upstream response fields to your shaped output
📤
Return shaped response to your app
Only the fields you defined in response_transform are returned

# Installation

bash
# Create a virtual environment
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

# Install dependencies
pip install flask flask-sqlalchemy httpx

# Optional: JMESPath for advanced path queries (items[0].id syntax)
pip install jmespath

Running the App

bash
python run.py

Server starts at http://localhost:5000. Landing page at /, Admin UI at /configs/ui.

# Database Migration

If upgrading an existing configs.db created before the transformation columns were added, run the migration script once:

bash
python migrate.py
ℹ️ For fresh installs, the database and all columns are created automatically on first startup via db.create_all(). No migration needed.

# Transformation System

All transformations use source expressions resolved against the incoming call payload.

Source Expressions

ExpressionBehaviourExample
fieldTop-level key lookup"user_id" → payload["user_id"]
parent.childDot-path traversal"user.email" → payload["user"]["email"]
items[0].idJMESPath (requires jmespath)First item's id from an array
$LITERAL:valueHardcoded constant string"$LITERAL:viewer" → "viewer"
$PASSTHROUGHSkip transform, use as-isSee body / response transform
💡 If jmespath is not installed, the system falls back to simple dot-path splitting. Install with pip install jmespath to unlock array and filter expressions.

Path Parameters

Inject dynamic values into URL placeholders using {param_name} syntax.

url template
https://api.example.com/orgs/{org_id}/users/{user_id}
path_params_schema
[
  { "name": "org_id",  "source": "org.id"  },
  { "name": "user_id", "source": "user.id" }
]

Payload { "org": { "id": "org_42" }, "user": { "id": "usr_7" } } resolves to https://api.example.com/orgs/org_42/users/usr_7

Query Parameters

Use value for static params and source for dynamic params resolved from the payload.

query_params_schema
[
  { "name": "limit",  "value":  "10"             },  // static
  { "name": "status", "source": "filter.status" },  // dynamic
  { "name": "page",   "source": "page"          }
]

Body Transform

Maps call payload fields to the outgoing upstream request body.

body_transform
{
  "username":    "user.name",
  "email":       "user.email",
  "role":        "$LITERAL:viewer",
  "external_id": "meta.id"
}

To forward the payload unchanged: { "$PASSTHROUGH": true }

⚠️ For GET, HEAD, and DELETE methods, the body is never sent regardless of this setting.

Response Transform

Maps upstream API response fields to your returned response. Source expressions are resolved against the upstream JSON body.

response_transform
{
  "userId":   "data.id",
  "fullName": "data.attributes.name",
  "email":    "data.attributes.email"
}

Omitting response_transform or using {"$PASSTHROUGH": true} returns the raw upstream response.

Header Transform

Dynamically inject request headers from the call payload at runtime. Ideal for multi-tenant systems.

header_transform
{
  "X-Tenant-Id":  "tenant_id",
  "X-User-Email": "user.email",
  "X-Request-Id": "meta.request_id"
}

Static Headers

Always-sent headers, regardless of call payload.

headers
{
  "Content-Type":  "application/json",
  "X-App-Version": "3.0"
}
ℹ️ Header merge order (highest priority wins): Static headers → Auth header → Header transform

# Auth Types

auth_typeBehaviourauth_value format
(omitted)No auth header added—
bearerAdds Authorization: Bearer <value>Raw token string
basicAdds Authorization: Basic <base64>username:password — encoded automatically
api_keyAdds <auth_header_name>: <value>Raw API key string

For api_key, set auth_header_name to override the default X-API-Key.

# Sample Payload

sample_payload is a stored JSON object used as the default for /sample and /dry-run endpoints. Override individual fields by including them in the request body — they are merged on top (request body wins).

bash
# Run with stored sample as-is
curl -X POST http://localhost:5000/execute/1/sample -H "Content-Type: application/json" -d '{}'

# Override one field
curl -X POST http://localhost:5000/execute/1/sample -H "Content-Type: application/json" -d '{"user_id": 99}'

# cURL Examples

All examples target http://localhost:5000. Replace IDs in execute calls with the id returned from create.

GET 1. Simple GET — No Auth
Plain passthrough with no transforms.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{"name": "get_public_ip", "url": "https://httpbin.org/get", "method": "GET", "enabled": true}'

# Execute
curl -s -X POST http://localhost:5000/execute/1 \
  -H "Content-Type: application/json" -d '{}'
GET 2. Path Param + Query Params
URL uses {user_id} placeholder; appends static and dynamic query params.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "get_user_posts",
    "url": "https://jsonplaceholder.typicode.com/users/{user_id}/posts",
    "method": "GET",
    "path_params_schema":  "[{\"name\": \"user_id\", \"source\": \"user_id\"}]",
    "query_params_schema": "[{\"name\": \"_limit\", \"value\": \"5\"}, {\"name\": \"_page\", \"source\": \"page\"}]",
    "sample_payload": "{\"user_id\": 1, \"page\": 2}",
    "enabled": true
  }'

# Dry run
curl -s -X POST http://localhost:5000/execute/2/dry-run \
  -H "Content-Type: application/json" -d '{"user_id": 3, "page": 1}'
POST 3. Bearer Token + Body Transform
Reshapes the call payload into the upstream request body and maps the response.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "create_user_bearer",
    "url": "https://httpbin.org/post",
    "method": "POST",
    "auth_type": "bearer",
    "auth_value": "my-secret-token-abc123",
    "body_transform": "{\"username\": \"user.name\", \"email\": \"user.email\", \"role\": \"$LITERAL:viewer\"}",
    "response_transform": "{\"sentBody\": \"json\", \"requestUrl\": \"url\"}",
    "sample_payload": "{\"user\": {\"name\": \"Jane Doe\", \"email\": \"jane@acme.com\"}}",
    "enabled": true
  }'
POST 4. Basic Auth
auth_value is username:password — base64-encoded automatically.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "basic_auth_post",
    "url": "https://httpbin.org/post",
    "method": "POST",
    "auth_type": "basic",
    "auth_value": "admin:supersecret",
    "body_transform": "{\"$PASSTHROUGH\": true}",
    "sample_payload": "{\"action\": \"sync\", \"target\": \"db\"}",
    "enabled": true
  }'
GET 5. API Key Header
Custom header name — useful for OpenWeatherMap, Stripe, etc.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "weather_api",
    "url": "https://httpbin.org/get",
    "method": "GET",
    "auth_type": "api_key",
    "auth_value": "abc123weatherkey",
    "auth_header_name": "X-API-Key",
    "query_params_schema": "[{\"name\": \"city\", \"source\": \"city\"}, {\"name\": \"units\", \"value\": \"metric\"}]",
    "sample_payload": "{\"city\": \"London\"}",
    "enabled": true
  }'
PUT 6. Dynamic Header Transform
Injects X-Tenant-Id and X-User-Email from the call payload at runtime.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "update_tenant_resource",
    "url": "https://httpbin.org/put",
    "method": "PUT",
    "auth_type": "bearer",
    "auth_value": "platform-level-token",
    "headers": "{\"Content-Type\": \"application/json\", \"X-App-Version\": \"2.1\"}",
    "header_transform": "{\"X-Tenant-Id\": \"tenant_id\", \"X-User-Email\": \"user.email\"}",
    "body_transform": "{\"resourceId\": \"resource.id\", \"payload\": \"resource.data\"}",
    "sample_payload": "{\"tenant_id\": \"acme-corp\", \"user\": {\"email\": \"ops@acme.com\"}, \"resource\": {\"id\": \"res_99\", \"data\": {\"status\": \"active\"}}}",
    "enabled": true
  }'

# Dry run — verify headers are resolved correctly
curl -s -X POST http://localhost:5000/execute/6/dry-run \
  -H "Content-Type: application/json" -d '{}'
DELETE 7. DELETE with Path Param
Delete a resource by resolving its ID from the call payload.
bash
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "delete_post",
    "url": "https://jsonplaceholder.typicode.com/posts/{post_id}",
    "method": "DELETE",
    "path_params_schema": "[{\"name\": \"post_id\", \"source\": \"post_id\"}]",
    "sample_payload": "{\"post_id\": 101}",
    "enabled": true
  }'
POST 8. Full Kitchen Sink
Path params + query params + bearer auth + static headers + header transform + body transform + response transform + sample payload.
bash — create
curl -s -X POST http://localhost:5000/configs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "full_kitchen_sink",
    "url": "https://httpbin.org/anything/{org_id}/users/{user_id}",
    "method": "POST",
    "auth_type": "bearer",
    "auth_value": "tok_live_xyz987",
    "headers": "{\"Content-Type\": \"application/json\", \"X-SDK-Version\": \"3.0\"}",
    "header_transform": "{\"X-Tenant\": \"org.slug\", \"X-Request-Id\": \"meta.request_id\"}",
    "path_params_schema": "[{\"name\": \"org_id\", \"source\": \"org.id\"}, {\"name\": \"user_id\", \"source\": \"user.id\"}]",
    "query_params_schema": "[{\"name\": \"expand\", \"value\": \"profile,roles\"}, {\"name\": \"locale\", \"source\": \"meta.locale\"}]",
    "body_transform": "{\"email\": \"user.email\", \"displayName\": \"user.name\", \"orgRole\": \"user.role\", \"source\": \"$LITERAL:api-wrapper\"}",
    "response_transform": "{\"echoedBody\": \"json\", \"requestUrl\": \"url\"}",
    "sample_payload": "{\"org\": {\"id\": \"org_42\", \"slug\": \"acme\"}, \"user\": {\"id\": \"usr_7\", \"name\": \"Ada Lovelace\", \"email\": \"ada@acme.com\", \"role\": \"admin\"}, \"meta\": {\"locale\": \"en-US\", \"request_id\": \"req_abc123\"}}",
    "enabled": true
  }'
bash — test
# Dry run
curl -s -X POST http://localhost:5000/execute/8/dry-run \
  -H "Content-Type: application/json" -d '{}'

# Execute with stored sample
curl -s -X POST http://localhost:5000/execute/8/sample \
  -H "Content-Type: application/json" -d '{}'

# Execute with live override
